关于 Git 的远程仓库

— — 此文用于千帆竞发项目远程仓库测试

基础指令

  • git remote add ( 远程主机名，统一用 origin，后文以 origin 代替 ) ( url ) => 与远程仓库建立连接

    git remote 指令 菜鸟教程

  • git fetch ( origin ) => 拉取远程仓库的更新，执行后需执行 merge 指令

    git fetch 指令 菜鸟教程

  • git merge ( origin )/( branch ) => 将远程仓库对应分支合并到本地目前分支

    git 分支管理 菜鸟教程

  • git pull ( origin ) ( main ):( brantest ) => 将远程主机 origin 的 master 分支拉取过来，与本地的 brantest 分支合并，如果远程分支是与当前分支合并，则冒号后面的部分可以省略

    git pull 指令 菜鸟教程

  • git push ( origin ) ( main ):( main ) => git push 命令用于从将本地的分支版本上传到远程并合并，如果本地分支名与远程分支名相同，则可以省略冒号

    git push 指令 菜鸟教程

  • git clone ( url 统一用 ssh ) => 将远程仓库内容克隆到本地并将与远程仓库建立连接

    git 远程仓库 菜鸟教程

初步测试流程

  • 选择一个合适的目录将项目克隆到本地

    1. 克隆 => 执行 git clone

  • 创建仓库

    1. 创建 => 执行 git init

  • 查看自己对应的分支（在命令提示符最右边），如果是 master 应改为 main（因为 GitHub 目前默认分支为 main），创建、切换、删除分支详见【git 分支管理】上面有链接

    1. 新建分支 => 执行 git branch ( 新分支名 )

    2. 切换分支 => 执行 git checkout ( 分支名 )

  • 在本地目录中添加一个文件，文件名包含自己的信息（不要有中文、特殊字符和空格）尝试暂存、提交到本地仓库，再更新到远程仓库

    1. 暂存 => 执行 git add ( filename )

    2. 提交 => 执行 git commit -m "( 提交信息 )"

    3. 更新到远程 => 执行 git push ( origin ) ( main )

最后

  • 刷新 GitHub 发现项目内增加了自己的文件即成功

  • 目前先完成这些操作，后面再测试分支与合并

  • 有问题发群里一起讨论

  • 文章可能有疏忽，发现请指正，一切以链接资料为准
