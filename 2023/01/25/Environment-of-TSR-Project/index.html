虚拟环境搭建——pipenv

pipenv 用于更加方便地管理项目依赖，可以为项目创建独立的虚拟环境，管理依赖

  • 安装 pipenv：pip install pipenv

  • 创建虚拟环境

    • pipenv install => 为项目创建一个文件夹，用来包含隔离的 python 解释器、依赖包。默认情况下，虚拟环境文件夹在 C:\Users\Admin\.virtualenvs\目录下创建

    • pipenv shell => 显式地激活虚拟环境，启动一个虚拟环境的子 shell，你会发现命令提示符前面添加了虚拟环境名称（退出使用 exit 命令）

  • 管理依赖

    • 如果创建虚拟环境时项目根目录没有 Pipfile 文件，使用 pipenv install 命令创建虚拟环境时，会自动在根目录下创建 Pipfile 和 Pipfile.lock，前者用来记录依赖列表，后者记录详细信息。当我们使用 pipenv install … 安装依赖时，Pipfile 和 Pipfile.lock 会自动更新。

    • 如果根目录内存在 Pipfile，使用 pipenv install 创建虚拟环境时会自动读取 Pipfile 文件并将依赖安装到创建的虚拟环境中。这样，当项目转移或部署上线时都很容易管理依赖。

工程结构

源码文件其中在 .\thousand_sails_race\ 中，其中：

  • templates\ 和 static\ 文件夹包含前端内容，html模板放在templates里面，css，js和其他图片等静态文件放在 static 中。

  • .py文件 主要是后端内容，视图函数放views.py中，表单类放forms.py中，数据库的类放models.py中，config内容放settings.py中。
